Helpful commands to analyze your elasticsearch cluster and How to interpret it.

Aug 15, 2016 #elasticsearch #cluster #monitoring

Sometimes, you don’t have access to GUI in your cluster for some reasons and surrendly you need to understand what’s going on, why your cluster are not responding so fast.

I recommend that you install Marvel or other 3rd party plugins, but is important to understand the data over just use it.

Elasticsearch, within your REST API, allows us to query the state of our cluster easily. It has two API:

  • Cat API
  • Cluster API

Cat API

It’s more well-formated output over Cluster API, (But doesn’t have all features that cluster api has).

In my opinion, the most important are:

  % curl 'localhost:9200/_cat/master?v&h=ip,port,heapPercent,name'
